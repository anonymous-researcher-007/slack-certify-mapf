# MovingAI benchmark suite

This directory holds the **MovingAI MAPF benchmark suite** used by
the §V experiments of the ASYU 2026 paper. Files are not committed
to git (they're large and have their own license — see
[LICENSE_movingai.md](LICENSE_movingai.md)); fetch them with the
download script below.

## Provenance

The benchmarks come from the [MovingAI](https://movingai.com/) MAPF
collection curated by Nathan Sturtevant and collaborators. If you use
them in academic work, please cite

> Stern, R., Sturtevant, N. R., Felner, A., Koenig, S., Ma, H.,
> Walker, T. T., Li, J., Atzmon, D., Cohen, L., Kumar, T. K. S.,
> Boyarski, E., and Bartak, R. *Multi-Agent Pathfinding: Definitions,
> Variants, and Benchmarks.* In Symposium on Combinatorial Search
> (SoCS), 2019.

A BibTeX block is included in [LICENSE_movingai.md](LICENSE_movingai.md).

## Maps included

| Map name                         | Family    | Dimensions | Description                          | Used in RQ |
|----------------------------------|-----------|------------|--------------------------------------|------------|
| `warehouse-10-20-10-2-1`         | warehouse | 161 × 63   | Small warehouse aisles               | RQ1, RQ2   |
| `warehouse-20-40-10-2-2`         | warehouse | 321 × 123  | Large warehouse aisles               | RQ1, RQ3   |
| `random-32-32-20`                | random    | 32 × 32    | 20 % random obstacles                | RQ2, RQ4   |
| `room-64-64-16`                  | room      | 64 × 64    | 16 × 16 rooms with bottlenecks       | RQ3, RQ5   |
| `maze-32-32-4`                   | maze      | 32 × 32    | 4-cell-wide corridors                | RQ4        |

Each map ships with **25 random scenarios** (`<map>-random-1.scen`
through `<map>-random-25.scen`) generated by the MovingAI tooling.

## How to download

```bash
bash benchmarks/download_movingai.sh
```

The script is idempotent — files already on disk with a matching
SHA-256 from [`checksums.sha256`](checksums.sha256) are skipped.
Failures are logged but do not abort the run, so partial benchmark
sets are usable for smoke tests.

After a successful run, refresh the checksum file with:

```bash
bash scripts/update_checksums.sh
```

## Checksums

[`checksums.sha256`](checksums.sha256) pins the SHA-256 of every
file the download script fetches. If a future MovingAI re-release
changes a file in place, the download script will report a hash
mismatch instead of silently overwriting.

## Adding more maps

To extend the registry to a new map:

1. Add an entry to `slackcertify.benchmarks.scenarios.ScenarioRegistry.MAP_FAMILIES`.
2. Add a row to the table above.
3. Add the URL pattern to `benchmarks/download_movingai.sh`'s `MAPS`
   array.
4. Re-run `scripts/update_checksums.sh`.
